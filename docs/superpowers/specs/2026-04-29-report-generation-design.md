# Phase 7b — Report Generation (PDF / HTML)

**Date:** 2026-04-29
**Project:** ArtifexProCal
**Scope:** Professional calibration report generation from session data, supporting PDF and HTML output with embedded visualizations.

---

## Goal

Enable users to generate professional-grade calibration reports from any session in their history. Reports include summary metrics, CIE diagrams, grayscale trackers, dE tables, LUT metadata, and before/after comparisons. Two formats: self-contained HTML (for sharing/email) and PDF (for archival/printing). Three templates cover common use cases.

## IPC Design

### Models

```rust
#[derive(Debug, Clone, serde::Serialize, specta::Type)]
pub struct ReportRequestDto {
    pub session_id: String,
    pub template: ReportTemplate,
    pub format: ReportFormat,
    pub compare_session_id: Option<String>,
}

#[derive(Debug, Clone, serde::Serialize, specta::Type)]
pub struct ReportResponseDto {
    pub path: String,
    pub format: ReportFormat,
}
```

### Command

```rust
#[tauri::command]
#[specta::specta]
pub async fn generate_report(
    service: State<'_, CalibrationService>,
    request: ReportRequestDto,
) -> Result<ReportResponseDto, String> {
    let detail = service.get_session_detail(&request.session_id).await
        .map_err(|e| e.to_string())?;
    
    let compare = if let Some(ref id) = request.compare_session_id {
        Some(service.get_session_detail(id).await.map_err(|e| e.to_string())?)
    } else {
        None
    };
    
    let bytes = ReportEngine::generate(
        request.template,
        request.format,
        &detail,
        compare.as_ref(),
    ).map_err(|e| e.to_string())?;
    
    let path = write_temp_file(&bytes, &request.format).map_err(|e| e.to_string())?;
    Ok(ReportResponseDto { path, format: request.format })
}
```

## Performance Considerations

- **SVG generation:** ~1-5ms per chart (simple SVG string building)
- **HTML rendering:** ~10-50ms for a full detailed report
- **PDF generation:** ~100-500ms depending on chart complexity (rasterization step)
- **File size:** HTML ~50-200KB (self-contained, no external assets). PDF ~100-500KB.

## Scope Exclusions

- **Custom report templates** — user-defined templates out of scope (Phase 10+)
- **Email sending** — just file generation; user handles distribution
- **Cloud-hosted reports** — local files only
- **Print dialog** — browser/Tauri print handles this for HTML
